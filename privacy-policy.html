5. 얼굴 데이터(Face Data) 수집 및 이용 / Face Data Policy

HAIRGATOR는 AI 헤어스타일 추천 및 시뮬레이션 기능을 위해 얼굴 데이터를 처리합니다.

5.1 수집 목적 (Reasons for Collection)

  • 한국어: 사용자의 얼굴 이미지를 수집하여 얼굴형, 이목구비 비율 등을 AI로 분석하고, 적합한 헤어스타일을 추천하며, 가상 헤어체험 시뮬레이션을 제공하기 위해 사용합니다.
  • English: We collect user's facial images to analyze face shape, facial features, and proportions using AI technology to recommend suitable hairstyles and provide virtual hairstyle simulation.

5.2 보유 기간 (Data Retention)

  • 한국어: 당사는 사용자의 원본 얼굴 데이터를 서버에 영구 저장하지 않습니다. 얼굴 이미지는 분석을 위해 일시적으로 처리되며, 결과 생성 직후 즉시 삭제됩니다. 처리 시간은 일반적으로 1분 이내입니다.
  • English: We do NOT store your raw facial data on our servers permanently. The facial image is processed transiently during the session for analysis and is deleted immediately after the recommendation/simulation is generated. Processing typically takes less than 1 minute.

Why this retention period: Face data is needed only for the brief moment of AI analysis. Since we do not need face data after generating results, immediate deletion protects user privacy and minimizes security risks.

5.4 제3자 제공 목적 (Reasons for Sharing)

  • Google Gemini API: Google의 생성형 AI 비전 기능을 활용하여 얼굴 이미지를 분석하고 스타일 추천 결과를 생성하기 위함입니다. (To utilize Google's generative AI vision capabilities to analyze the image and generate style recommendations.)
  • vModel API: AI 기반 가상 헤어스타일 합성 기술을 활용하여 시뮬레이션 이미지를 생성하기 위함입니다. (To generate simulation images using AI-based virtual hairstyle synthesis technology.)

5.5 제3자의 데이터 처리 (Third-Party Data Storage Practices)

Google LLC (Gemini API)

  • 저장 여부 (Does Google store face data?): 아니오. Google Gemini API는 API 요청에 포함된 이미지 데이터를 영구 저장하지 않습니다.
    No. Google Gemini API does NOT permanently store image data included in API requests.
  • 보유 기간 (Retention period): 0초 - API 응답 생성 직후 즉시 삭제됩니다.
    0 seconds - deleted immediately after generating the API response.
  • 보유 이유 (Why this retention period): Google은 API 요청을 처리하는 동안에만 일시적으로 데이터를 메모리에 보관합니다. 응답이 반환되면 더 이상 데이터가 필요하지 않으므로 즉시 삭제됩니다.
    Google only temporarily holds data in memory while processing the API request. Once the response is returned, the data is no longer needed and is immediately discarded.
  • AI 학습 사용 (Used for AI training?): 아니오. Google Cloud API 서비스 약관에 따라 API 요청 데이터는 모델 학습에 사용되지 않습니다.
    No. Per Google Cloud API Terms of Service, API request data is not used for model training.
  • 개인정보 처리방침: https://policies.google.com/privacy

vModel Inc. (vModel API)

  • 저장 여부 (Does vModel store face data?): 예, 일시적으로. 이미지 처리 큐 및 결과 전달을 위해 최대 24시간 보관합니다.
    Yes, temporarily. Images are retained for up to 24 hours for processing queue and result delivery.
  • 보유 기간 (Retention period): 최대 24시간
    Up to 24 hours maximum
  • 보유 이유 (Why this retention period): vModel의 AI 이미지 생성 시스템은 비동기 작업 큐를 사용합니다. 사용자가 결과를 다운로드할 수 있도록 생성된 이미지를 일시적으로 보관해야 합니다. 24시간은 사용자가 결과를 확인하기에 충분하면서도 개인정보 보호를 위해 최소한으로 설정된 기간입니다. 24시간 후 원본 얼굴 이미지와 생성된 결과물 모두 자동 삭제됩니다.
    vModel's AI image generation system uses an asynchronous job queue. Generated images must be temporarily stored so users can download results. 24 hours provides sufficient time for users to retrieve results while being the minimum period necessary for privacy protection. After 24 hours, both the original face image and generated results are automatically deleted.
  • AI 학습 사용 (Used for AI training?): 아니오. vModel은 사용자가 업로드한 이미지를 AI 모델 학습 목적으로 사용하지 않습니다.
    No. vModel does NOT use user-uploaded images for AI model training purposes.
  • 개인정보 처리방침: https://vmodel.ai/privacy

Summary: Neither third party stores face data indefinitely. Google deletes immediately (0 seconds). vModel retains for maximum 24 hours for technical processing needs, then automatically deletes. Neither party uses face data for AI model training.

  • Face data upload is entirely optional. Users can use other features of the service without uploading facial photos.

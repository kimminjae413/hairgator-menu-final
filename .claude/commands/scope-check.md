# 변경 범위 사전 확인

작업을 시작하기 전에 다음을 확인하고 승인을 받습니다.

## 1. 수정할 파일 목록

| 파일 | 수정 내용 | 유형 |
|------|----------|------|
| (파일 경로) | (무엇을 수정할지) | CSS/JS로직/HTML구조 |

## 2. 변경 유형 분류

- [ ] **CSS/스타일만**: 색상, 폰트, 레이아웃, 애니메이션
- [ ] **HTML 구조**: 태그 추가/삭제/변경
- [ ] **JS 로직**: 함수 동작, 이벤트 처리, API 호출

## 3. 영향 범위 체크

- [ ] 이 변경이 다른 기능에 영향을 줄 수 있는가?
- [ ] 기존에 잘 동작하던 것이 망가질 가능성이 있는가?
- [ ] 테스트가 필요한 부분이 있는가?

## 4. 사용자 확인 요청

위 내용을 확인하시고, 진행해도 되면 "진행"이라고 답변해주세요.

수정 범위가 예상과 다르면 알려주세요.

---

**⚠️ JS 로직 변경이 포함된 경우:**
- 어떤 함수가 영향받는지 명시
- 기존 동작이 어떻게 바뀌는지 설명
- 사이드 이펙트 가능성 안내
